# OpenSkills 技能优先级和创建顺序

## 技能总览（20 个）

基于项目实际开发流程分析，按优先级和使用频率排序。

## 优先级说明

- ⭐⭐⭐⭐⭐：日常开发必备，每天都会用到
- ⭐⭐⭐⭐：高频操作，每周多次使用
- ⭐⭐⭐：专项功能，每周 1-2 次
- ⭐⭐：辅助功能，偶尔使用
- ⭐：特殊场景，很少使用

## 第一批：核心技能（必须创建）⭐⭐⭐⭐⭐

### 1. monorepo-quick-start
- **覆盖场景**：项目结构、应用列表、包依赖、快速定位
- **使用频率**：每天多次
- **创建难度**：简单
- **预计时间**：1-2 小时

### 2. dev-workflow
- **覆盖场景**：启动开发服务器、热重载、端口配置、微前端联调
- **使用频率**：每天多次
- **创建难度**：简单
- **预计时间**：1-2 小时

### 3. build-guide
- **覆盖场景**：3 种构建模式、单应用/全量构建、预览构建
- **使用频率**：每天 1-3 次
- **创建难度**：中等
- **预计时间**：2-3 小时

### 4. scripts-navigator
- **覆盖场景**：Scripts 新架构导航、脚本分类、常用脚本速查
- **使用频率**：每天多次
- **创建难度**：简单
- **预计时间**：1-2 小时

**第一批小计**：6-9 小时，覆盖 80% 日常场景

## 第二批：高频技能（强烈推荐）⭐⭐⭐⭐

### 5. release-automation
- **覆盖场景**：全自动发布、CHANGELOG 更新、Git 标签管理
- **使用频率**：每周 1-2 次
- **创建难度**：中等
- **预计时间**：2-3 小时

### 6. i18n-toolkit
- **覆盖场景**：翻译检查、合并、新增、验证
- **使用频率**：每周 2-3 次
- **创建难度**：中等
- **预计时间**：2-3 小时

### 7. deploy-toolkit
- **覆盖场景**：4 种部署方式、环境配置、部署测试
- **使用频率**：每周 2-3 次
- **创建难度**：复杂
- **预计时间**：3-4 小时

### 8. quality-assurance
- **覆盖场景**：Lint、Type-check、循环依赖检查、测试
- **使用频率**：每天 1-2 次
- **创建难度**：中等
- **预计时间**：2-3 小时

**第二批小计**：9-13 小时，覆盖额外 15% 场景

## 第三批：专项技能（按需创建）⭐⭐⭐

### 9. component-development
- **覆盖场景**：共享组件开发、命名规范、导出引用
- **使用频率**：每周 1-2 次
- **创建难度**：中等
- **预计时间**：2 小时

### 10. module-system
- **覆盖场景**：模块扫描、配置规范、路由自动发现
- **使用频率**：每周 1 次
- **创建难度**：中等
- **预计时间**：2 小时

### 11. troubleshooting
- **覆盖场景**：常见错误诊断、构建失败、运行时错误
- **使用频率**：每周 1-2 次
- **创建难度**：复杂
- **预计时间**：3-4 小时

### 12. docs-system
- **覆盖场景**：VitePress 文档编写、部署、配置
- **使用频率**：每周 1 次
- **创建难度**：简单
- **预计时间**：1-2 小时

### 13. git-workflow
- **覆盖场景**：分支策略、Commit 规范、PR 流程
- **使用频率**：每天 1 次
- **创建难度**：简单
- **预计时间**：1-2 小时

**第三批小计**：9-12 小时

## 创建顺序建议

### 快速验证路径（2-3 天）

```
Day 1（上午）：
1. 安装 OpenSkills（30 分钟）
2. 创建 monorepo-quick-start（1.5 小时）
3. 创建 dev-workflow（1.5 小时）

Day 1（下午）：
4. 创建 build-guide（2.5 小时）
5. 创建 scripts-navigator（1.5 小时）

Day 2：
6. 测试和优化前 4 个技能（4 小时）
7. 实际使用并收集反馈

Day 3-14：
8. 实际开发中验证效果
9. 根据需要调整和补充
```

### 全面实施路径（1-2 周）

```
Week 1：
- Day 1-2：第一批（4 个核心技能）
- Day 3-4：第二批（4 个高频技能）
- Day 5：第三批前 3 个

Week 2：
- Day 1：第三批后 2 个
- Day 2：第四批（5 个辅助技能）
- Day 3：第五批（2 个特殊场景）
- Day 4-5：测试、优化、文档完善
```

## 技能创建模板参考

### 简单技能（1-2 小时）

结构：
```
.claude/skills/skill-name/
└── SKILL.md（纯指令，无额外资源）
```

内容包含：
- 何时使用
- 基本操作步骤
- 常用命令列表
- 注意事项

### 中等技能（2-3 小时）

结构：
```
.claude/skills/skill-name/
├── SKILL.md
└── references/
    └── quick-reference.md（快速参考）
```

内容包含：
- 详细操作流程
- 多种场景处理
- 配置文件说明
- 常见问题解答

### 复杂技能（3-4 小时）

结构：
```
.claude/skills/skill-name/
├── SKILL.md
├── references/
│   ├── architecture.md
│   └── advanced-usage.md
├── scripts/
│   └── helper.mjs
└── assets/
    └── templates/
```

内容包含：
- 完整工作流程
- 架构说明
- 辅助脚本
- 模板文件
- 高级用法
- 故障排查

## 维护和更新计划

### 日常维护

- **频率**：每周检查 1 次
- **内容**：根据项目变更更新技能内容
- **责任人**：技能创建者或指定维护人

### 版本更新

- **触发条件**：项目架构重大变更（如 scripts 重构）
- **内容**：更新相关技能的指令和参考
- **流程**：
  1. 识别受影响的技能
  2. 更新 SKILL.md
  3. 测试验证
  4. 提交到版本控制

### 废弃机制

- **条件**：功能被移除或完全改变
- **处理**：移动到 `.claude/skills/archive/` 或直接删除
- **文档**：在 CHANGELOG 中说明
