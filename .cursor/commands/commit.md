# 快速提交到 GitHub develop 分支

请执行以下操作快速提交当前项目的所有更改到 GitHub 的 develop 分支：

## 执行步骤

1. **检查工作目录**
   - 确认当前在项目根目录（`btc-shopflow-monorepo`）

2. **检查变更状态**
   - 使用 `git status --porcelain` 检查是否有未提交的更改（**不要使用 `git diff`，会导致命令卡住**）
   - 使用 `git status` 查看变更的文件列表和详细信息

3. **生成提交消息**（如果用户未提供）
   - 分析变更的文件列表和修改类型（新增、修改、删除）
   - 根据变更内容自动生成合适的提交消息
   - 提交消息格式：`type: description`，例如：
     - `feat: 添加新功能`
     - `fix: 修复bug`
     - `chore: 更新配置`
     - `docs: 更新文档`
     - `refactor: 重构代码`
     - `style: 代码格式调整`
   - 提交消息应该简洁、准确地描述本次变更的主要内容

4. **切换到 develop 分支**（如果当前不在）
   - 执行 `git checkout develop`

5. **添加所有更改**
   - 执行 `git add .`

6. **提交更改**
   - 执行 `git commit -m "生成的提交消息"`（使用自动生成的或用户提供的提交消息）

7. **推送到远程**
   - 执行 `git push origin develop`

8. **显示结果**
   - 显示提交结果和相关信息
   - 显示使用的提交消息

## 重要提示

- ❌ **禁止使用 `git diff`**：会导致命令卡住
- ✅ **使用 `git status`**：查看变更信息以生成提交消息
- ✅ **自动生成提交消息**：根据实际变更内容生成，而不是使用固定默认值
- ✅ **提交消息要准确**：应该反映实际的代码变更内容
